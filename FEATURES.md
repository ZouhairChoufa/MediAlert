### üîå API Features

#### Authentication
- API key-based authentication
- Secure token management
- Rate limiting per key
- Request validation

#### Endpoints

##### Alert Management
```
POST /api/alert
- Create new emergency alert
- Required: symptoms, location
- Optional: name, age, gender
- Returns: patient ID and status
```

##### Patient Data
```
GET /api/patients
- List all patients
- Returns: array of patient objects
- Includes: demographics, status, assignments

GET /api/patient/{id}
- Get specific patient details
- Returns: complete patient record
- Includes: ambulance, hospital, medical team
```

##### Fleet Management
```
GET /api/ambulances
- Get ambulance fleet status
- Returns: array of ambulance objects
- Includes: ID, status, location, availability
```

##### Hospital Network
```
GET /api/hospitals
- Get hospital network information
- Returns: array of hospital objects
- Includes: name, beds, status, capabilities
```

#### Response Format
- **Success Response**
```json
{
  "status": "ok",
  "data": { ... },
  "timestamp": "2024-01-01T12:00:00Z"
}
```

- **Error Response**
```json
{
  "status": "error",
  "message": "Error description",
  "code": 400
}
```

#### Rate Limiting
- 1000 requests per hour per API key
- Rate limit headers in response
- 429 status code when exceeded
- Automatic reset after time window
